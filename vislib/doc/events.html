Handler

class Handler{
public:
  virtual void mousehandler( const MouseEvent& ) = 0;
  virtual void keyhandler( const KeyEvent& ) = 0;
  virtual void tick( fpnum tcks ) = 0;
};

Handler is an abstract class that represents the functions necessary for running a program. It has three virtual methods: tick, mousehandler and keyhandler. mousehandler and keyhandler get called during an Engine::tick if there is pending input. The handler tick gets called every time Engine::tick is called. Every time the handler tick gets called its parameter will be set to the approximate number of milliseconds since the last call to Engine::tick.

MouseEvent

struct MouseEvent{
  static const byte leftbutton = 1;
  static const byte rightbutton = 2;
  static const byte middlebutton = 3;
  enum Type { pressed, released, doubleclicked } type; 
  byte button;
};

A MouseEvent has two fields, type, which is an enumerant that tells whether it was a press, a release or a double click and button, which indicates the button that generated the event. MouseEvent defines three constants: leftbutton, rightbutton and middlebutton, these are the possible values of button when a event is sent.

KeyEvent

struct KeyEvent{
  bool pressed;
  byte symbol;
};

A KeyEvent has two fields. The first, pressed, is true if the event is a key press and false if it is a key release. The second, symbol, is the symbol for the key that got pressed. The symbol is a VK type symbol, such as VK_ESCAPE.
